Patient's login

Data is sent once the patient logs in

Property Description
email Patient's first option identifier to his account
phoneNumber Patient's second option identifier to his account
password The password of the patient's account

POST /api/login

Example Request

{
    "email": "patient@example.com",
    "password": "patientpassword"
}

Example Response

{
    "token": "your_token_here",
    "patient_id": "user"
}

Example Error

{
    "status": "error",
    "error": "invalid credentials"
}
